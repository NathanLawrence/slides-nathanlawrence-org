Asynchronous Execution is Beautiful and Terrible

  • Non-blocking code makes life magical.
  • It helps our users get what they want faster.
  • It helps us create more semantically appropriate code.
  • It embraces the inherent need for constant updates in 21st century apps.

But it can get sooooo ugly soooooooo fast.

The Typical Story of an AJAX Request

  1. Go get some information for me.
  2. Once you have that information, perform some basic processing on it.
  3. If you need more, go get that additional information and process it.
  4. Execute whatever display logic is necessary.

Plus, if anything goes wrong along this path, handle that event appropriately.

What if, instead chaining code that handles data when it comes in, we write code that acts on the idea that data will arrive at some point?
